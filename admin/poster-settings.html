Setup Supabase — Istruzioni (clicca per espandere)

Per utilizzare questa pagina, assicurati di aver eseguito il seguente setup nel tuo progetto Supabase.

1. Crea la tabella settings

Incolla questo SQL nell'editor SQL di Supabase Dashboard:

-- Crea la tabella settings
CREATE TABLE IF NOT EXISTS public.settings (
  id          bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  key         text UNIQUE NOT NULL,
  value       jsonb,
  updated_at  timestamptz DEFAULT now()
);

-- Abilita RLS
ALTER TABLE public.settings ENABLE ROW LEVEL SECURITY;

-- Policy: solo utenti autenticati possono leggere/scrivere
CREATE POLICY "Admin read settings"
  ON public.settings FOR SELECT
  USING (auth.role() = 'authenticated');

CREATE POLICY "Admin write settings"
  ON public.settings FOR ALL
  USING (auth.role() = 'authenticated')
  WITH CHECK (auth.role() = 'authenticated');

2. Crea il bucket Storage posters

  1. Vai su Supabase Dashboard → Storage
  2. Clicca "New bucket"
  3. Nome: posters
  4. Spunta "Public bucket" (così i partner possono scaricare il template)
  5. Clicca Save
-- Oppure via SQL (richiede permessi di storage admin):
INSERT INTO storage.buckets (id, name, public)
VALUES ('posters', 'posters', true)
ON CONFLICT (id) DO NOTHING;
